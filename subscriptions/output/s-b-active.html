Active Status

Description

This subscription status indicates that a Vendor successfully approved a fulfillment request. Therefore, a Vendor provides required data (like product activation keys or URLs) to a consumer and the system assigns the Active status to the subscription. An active subscription can be cancelled; thus, the system assigns the Terminating status to this subscription. Once a subscription is activated, consumers might also create a change request to adjust items or item quantity.
Furthermore, an active subscription can be suspended in case a corresponding capability is enabled.

Prerequisites

A subscription with the Processing status and an approved fulfillment request.

Transferable statuses

In case a cancel request is created, the active subscription is transferred to the Terminating state. It is also possible to transfer active subscriptions to the Suspended state in case Administrative Hold capability is enabled.

Associated transitions

  • Change Request Creation
  • Change Request Approval
  • Subscription Suspension
  • Subscription Resumption
  • Subscription Cancellation
